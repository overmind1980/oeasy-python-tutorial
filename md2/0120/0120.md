### copy()

- 使用copy方法
	- 可以 分别存储 

```

man[0] = "o2z"
print(clone)
```

- 两个列表

	- 能够 独立修改

### 多维列表

### 深拷贝

- 浅拷贝(shallow copy)
	- shallow = 浅的，表面的
		- 只拷贝 最浅层的
		- 浅浅的拷贝
		- 不申请 新的内存空间
- `浅拷贝` 不行
	- 有 `深拷贝` 吗？

### 具体操作

- 有深浅变化
	- 才有意思啊

```
import copy
man = ["oeasy",[165,70]]
clone = copy.deepcopy(man)
clone[1][0] = 65
```

- 申请了 新空间

![图片描述](uid1190679-20260307-1772835221111.png) 

- 深拷贝 能有多`深` 呢？？

###  知深浅

- 来个深的

```python
import copy
l1 = [1, [2, [3, 4, [5, 6]]]]

```

- 这么深的话
	- 深拷贝 都能 都复制出来 么？

### 深拷贝的威力

- 深拷贝 
	- 讲究一个 `深` 字
	- 要多深 有多深 

![图片描述](uid1190679-20250115-1736931842506.png) 

- 完全独立的 空间
	- 互不影响 

### 赋值、拷贝总结

- 等号赋值
	- 完全共享
	- 一改全改

```
l2 = l1
```

- 浅拷贝 copy()
  - 浅层独立
  - 深层共享

```
l2 = l1.copy()
```

- 深拷贝 deepcopy
	- 全新空间
	- 彻底独立

```
l2 = copy.deepcopy(l1)
```

- 内存的状态 
	- 会 影响到 方方面面

### 新情况

```python

x[0].append(1)
```

- 这和刚才
	- 完全不同

![图片描述](uid1190679-20260307-1772835743200.png) 

- 三个列表项
	- 存在 三个地方

- 用deepcopy
	- 可以吗？

### 乘法

```
from copy import deepcopy
x = [[0]]
x = deepcopy(x) * 3
x[0].append(1)
```

- 乘法 不行

![图片描述](uid1190679-20260307-1772835861416.png) 

- 因为 同一个地址 乘了三次

### 加法

```
from copy import deepcopy
x = [[0]]
x = deepcopy(x) + deepcopy(x) + deepcopy(x) 
x[0].append(1)
```

- 加法可以
	- 每个 都是 深拷贝 出来的 副本

![图片描述](uid1190679-20260307-1772835938027.png) 

- 每个 加数 
	- 都分配了 新空间 

### 总结

- 赋值的 三种情况 

| 赋值方式 | 核心特点（大白话口诀） |
|----------|------------------------|
| `=`      | 共用地址，一改全改     |
| `copy()` | 浅层独立，深层共用     |
| `deepcopy()` | 全部独立，互不干扰 |
