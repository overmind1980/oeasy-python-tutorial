### 地址

- 英雄列表

```
hero_list = ["刘备", "关羽", "张飞"]
print(hero_list)
del hero_list[1:2]
print(hero_list)
```

- 帧栈(Frame)上 只能看见 hero_list
	- hero_list 存的是 引用的地址(id)

### 互相影响

- 直接 用列表 给 新变量赋值
	- 让clist1、clist2指向同一个地址空间

```
clist1 = list("oeasy")
clist2 = clist1
clist1.append("?")
clist2.append("!")
```

![图片描述](uid1190679-20251106-1762394100104.png) 

- 为啥clist1、clist2会相互影响？

### id

- 去看看 具体地址

```
clist1 = list("oeasy")
print(clist1,id(clist1))
clist2 = clist1
print(clist2, id(clist2))
clist1.append("?")
clist2.append("!")
print(id(clist1) == id(clist2))
```

- clist1、clist2 引用的是
	- `同一个`对象地址
	- 所以 
		- 不论 谁append
		- 都会append到 这个地址对应的列表 上

![图片描述](uid1190679-20251106-1762394189176.png) 

- 列表 被引用了 `几次` 呢？

### getrefcount

- getrefcount 
	- 可以得到  对象
	- `引用次数`

```
import sys
clist1 = list("oeasy")
print(sys.getrefcount(clist1))
```

- 明明是 clist1 
	- 1个变量引用

![图片描述](uid1190679-20251107-1762480567454.png) 

- 为什么 显示 `2个引用` 呢？

### 帮助手册

```
import sys
help(sys.getrefcount)
```

- 每次 getrefcount被调用时
	- 增加 1个引用
	- 所以显示2个

![图片描述](uid1190679-20210905-1630830139942.png)

- 取消引用
	- 被引用数 会减少 吗？

### 取消引用

```
clist2 = clist1
print(sys.getrefcount(clist1))
clist2 = []
print(sys.getrefcount(clist1))
del clist1
print(sys.getrefcount(clist1))
```

- 这时候 clist2引用 别的地址
	- 那clist1 引用数 就会减1

![图片描述](uid1190679-20251107-1762480750279.png) 

- clist1 和 clist2 引用 `同一`位置
	- 他俩 会`一改全改`
	- 有什么办法 不要 `一改全改` 吗？

### 使用构造函数 

- 新作一个列表
	- 根据 nlist1
	- 再`新`造一个 nlist2

```python
nlist1 = [1, 2, 3]
nlist2 = list(nlist1)
print(id(nlist1), id(nlist2))
```

- id `不同`
	- 对应的空间 也`不同`

![图片描述](uid1190679-20251107-1762501044502.png) 

- 还有 啥方法 可以`不影响` 原列表 吗？

### copy副本

- 使用list类的 copy方法

```
clist1 = list("oeasy")
clist2 = clist1.copy()
clist1.append("?")
clist2.append("!")
```

- 列表 和 列表的拷贝 
	- 引用不同的位置

![图片描述](uid1190679-20210829-1630210778781.png)

- 这 两个列表 地址还相同 吗？

### 确保

- 观察 id
	- https://pythontutor.com/render.html#mode=display

```
clist1 = list("oeasy")
clist2 = clist1.copy()
print(id(clist1))
print(id(clist2))
```

- id不同
	- copy后 就新分出 空间

![图片描述](uid1190679-20251107-1762502444744.png) 

- 给 原列表 做了个副本

### 总结🤔

- 列表 赋值运算 两种形式
  1. 将列表 直接 赋值
		- 造成两个列表指向同一个对象
		- 一改全改
  2. 将 列表副本 赋给 变量
		- 这两个列表变量指向不同的对象
		- 互不影响

```
clist1 = list("oeasy")
clist2 = clist1
clist2 = clist1.copy()
```
