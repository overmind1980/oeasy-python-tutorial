### 字典更新

- 字典的合并使用 update
	- 两本字典放在一起
	- 合成一本字典
	- union 并集

```python
d1 = {"a": 1}
d2 = {"b": 2}
d1.update(d2)
d1
```

- 字典 是 把参数字典
	- 合并过来
	- 让字典添加 新字典项

![图片描述](uid1190679-20210914-1631608478627.png)

- 但是如果 两个字典
	- 同一key的值 `不`一致
    - 会如何呢?

### 字典更新

- 试试

```python

```

- 以后面的为准吗

### 快速写法

```python
d1 = {"a": 1, "b": 2}
d2 = {"a": 2, "c": 3}
merge_dict = {**d1, **d2}
merge_dict
d1
d2
```

- 效果

![图片描述](uid1190679-20260316-1773666580869.png) 

- 这 **d1 怎么理解？

### 字典解包

- **d1 是 字典解包
	- **d2 也是 字典解包

```
merge_dict = {**d1, **d2}
```
- 把两个 解包后的字典 
	- 组合成 新字典merge_dict
	- d1、d2都没有改变
- 除了 update
	- 还有一个 setdefault 命令
	- 也可以 添加字典项

### 默认 default

- 查看帮助

```
help(dict.setdefault)
```

- 如果 没这个key
	- 就 插入这个key
	- 并返回这个默认值

![图片描述](uid1190679-20210914-1631609246349.png)

- 如果 有这个key
	- 再想 设置新的值 是不行的
	- 返回 原来的默认值

### 实践

```python
d = {}
d.setdefault("name", "oeasy")
```

### 赋值


```python
d = dict(name = "oeasy")
d
```

- 有两种方式 强制更新字典
	1. update
	2. 通过key索引后 赋值

```python
d.update(name = "o2z")
d
d["name"] = "o3z"
d
d.setdefault("name", "o4z")
d
```

- 这两种设置
	- 和setdefault不同
	- setdefault是 `试着设置` 字典项

![图片描述](uid1190679-20251112-1762923696683.png)

### default

```python

```

- get 是 试着来
