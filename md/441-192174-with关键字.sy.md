### PEP

- Python Enhancement Proposals
- python 增强改进建议
- 主要问题就是如果写文件打开了
- 然后写的时候出了问题
- 但是报了错之后的 close 函数就不执行了
- 原来用一种 try...finally...的结构
- 保证 close 在 finally 里面永远执行
- 用 with 的方法
- 保证 close 在最后永远执行
- 而且写的还简单
- python 是不会随意增加关键字的
- 就连 int、str 这些都是类而不是关键字
- 看来 with 确实必要
- 还有什么PEP呢？
